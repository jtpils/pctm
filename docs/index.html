Title: Point Cloud to Mesh

Summary

In this project, we will implement algorithms for reconstructing 3D surfaces (meshes) from point clouds. This type of conversion is useful as many 3D scanning technologies capture data in the point cloud format, and 3D meshes provide for more intuitive interpretation and manipulation in graphics applications.

Problem Description

The general problem that this project addresses is the question of how to capture the 3D geometry of some object in the real world, and then transform the data into a useful digital representation. Today, this is commonly accomplished via 3D scanning technologies such as coordinate measuring machines, laser scanners, or ultrasound scanners. Typically, these technologies work by capturing the geometry of an object by sampling depth at certain 2D coordinates along a certain plane , then combining the depth information with the 2D coordinates to form 3D points. This process results in a point cloud, which is essentially just the raw collection of points. While this type of representation may be good enough in order to interpret the geometry of the captured object, transforming point clouds into surface representations such as meshes is often useful for further manipulation of the data in CAD applications, for instance.

This problem can be challenging to solve in the sense that point clouds do not inherently convey any sense of orientation or connectivity. In addition, because collecting point cloud data is a sampling process, it is susceptible to noise and aliasing. Therefore, good solutions must be able to robustly handle these issues. In our implementations, we intend to deduce surface information by approximating alpha shapes (generalizations of the convex hull), and surface indicator functions (3D functions that are 1 at points on the interior of a surface and 0 elsewhere).

Goals and Deliverables

Because the goal of this project is to reconstruct 3D meshes, we naturally intend to deliver example reconstructions in the form of images (png) and mesh (dae) files. To generate these, we will use our reconstruction implementations together with MeshEdit, which provides infrastructure to draw and screen capture the surface of a mesh.

In order to gauge the quality of our implementation, we will test it on benchmark meshes such as the Stanford bunny and dragon that we have already seen in this class from Project 3. In addition to comparing these to the provided mesh reconstructions from the Stanford repository, we will compare our reconstructions with other reconstructions generated by ready-made point cloud converters available for download online. By comparing against other peopleâ€™s implementations using the same algorithm, we can obtain a sense of the accuracy and correctness of our own implementation.

Achievable Deliverables

For this project, we strongly intend (plan) to deliver a working implementation of the ball-pivoting surface reconstruction algorithm described here, along with the accompanying files for comparison as described above. We believe this should be feasible to achieve as the ball-pivoting algorithm is described as relatively simple to understand and implement compared to other surface reconstruction algorithms.

Ambitious Deliverables

In the case that we do well (quality and time-wise) with the ball-pivoting algorithm, we will attempt to implement more state-of-the-art algorithms such as Poisson Surface Reconstruction, or Voronoi-Based Variational Reconstruction of Unoriented Point Sets. The deliverables for these would be the same as for ball-pivoting, just achieved with a different algorithm.
